# Data Structures & Algorithms

This section focuses on the coding aspect of the interview, which is arguably the most critical for entry to mid-level roles (L3/L4) at Google.

## Key Areas to Master

Google interviews often test your ability to combine multiple concepts. Make sure you are strong in the following areas:

### 1. Graphs and Trees
- Depth-First Search (DFS) & Breadth-First Search (BFS)
- Topological Sorting
- Dijkstra's Algorithm
- Trie (Prefix Tree)
- Common patterns: Matrix traversals, finding shortest paths, cycle detection.

### 2. Dynamic Programming
- 1D and 2D DP
- Knapsack variants
- Longest Common Subsequence / Substring
- **Google Tip:** DP is common, but don't jump to it immediately. Often, a greedy approach or basic recursion with memoization is sufficient.

### 3. Arrays, Strings, and Maps
- Sliding Window
- Two Pointers
- Prefix Sums
- Hashing (HashMaps, HashSets) - Extremely crucial for optimizing O(n^2) to O(n).

### 4. Advanced Concepts (Good to know)
- Union Find (Disjoint Set)
- Segment Trees (Rare but possible for L5+)
- Bit Manipulation

## Recommended Study Plan
1. **Understand the Theory:** Ensure you know the time and space complexity of every major data structure and algorithm.
2. **Pattern Recognition:** Don't just solve problems; understand the *patterns* (e.g., merging intervals, two heaps, sliding window).
3. **Practice Constraints:** Try to solve easy problems in 10-15 mins, mediums in 25-30 mins, and hards in 40-45 mins.
