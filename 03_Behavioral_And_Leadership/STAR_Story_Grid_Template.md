# STAR Method Story Grid

Use this grid to prepare your behavioral stories. Ideally, you want 5-7 strong, versatile stories that can be adapted to answer various behavioral questions.

| Theme | Common Questions | Story Title (Brief) | S (Situation) | T (Task) | A (Action) | R (Result) | Applicable Values |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Conflict** | Tell me about a time you disagreed with a coworker. | *The Database Migration Disagreement* | Team couldn't agree on SQL vs NoSQL for the new microservice. | Had to reach consensus before project deadline. | Organized a bake-off. Built POCs for both. Presented metrics on latency and scale. | Team universally agreed on NoSQL based on data. | Data-driven, Leadership, Collaboration |
| **Failure** | Tell me about a time a project failed. |  |  |  |  |  |  |
| **Leadership** | Tell me about a time you led without authority. |  |  |  |  |  |  |
| **Ambiguity** | Tell me about a time you lacked information to proceed. |  |  |  |  |  |  |
| **Innovation** | Tell me about a time you improved a process. |  |  |  |  |  |  |

## Tips:
- **Keep it concise:** The 'S' and 'T' should take up ~20% of your time. The 'A' and 'R' should be ~80%.
- **Highlight your specific impact:** Use "I" instead of "We" when describing actions.
- **Quantify Results:** "Reduced latency by 20%" is better than "Made it faster".
